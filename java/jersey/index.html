For non-maven project, best to use the list of dependencies.

For a maven project: 


    org.glassfish.jersey.ext

    2.12

Encoding (Jersey 2)

It seems Jersey 2 sets an encoding content type header as encoded in ISO-8859-1, even if the content is encoded in UTF-8 and a JSP page directive sets the proper value. The solution is to use servlet filter to override the behavior. Tomcat provides one out of the box, ready to use (configuration only, no coding).
